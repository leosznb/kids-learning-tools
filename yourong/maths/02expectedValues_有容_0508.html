Part A: Expected Number of Occurrences預期發生次數

Expected number = n × p

預期次數 = 試驗次數 × 概率

If the probability of an event is p, then after n trials, we expect this event to occur n × p times.
如果一個事件每次發生的概率是 p，重複試驗 n 次後，我們預期它會發生 n × p 次。

1
Find the probability p of the event.
先找出事件發生的概率 p。
2
Multiply: n × p.
然後乘以試驗次數 n。
💡
The expected number may NOT be an integer. (e.g. 14.8 times — it's an average, not a count.)
預期次數不一定是整數。（例：14.8 次 —— 這是平均值，不是實際次數。）

Example 1 (from textbook): A fair dice is thrown 30 times. Find the expected number of times of getting a '2' or a '3'.
擲一粒公平骰子 30 次，求擲到「2」或「3」的預期次數。

1
P(getting a '2' or '3') = (1 + 1) / 6 = 2/6 = 1/3
2
Expected number = 30 × (1/3) = 10

That was Part A — simple and quick. Now the main event.
Part A 就這麼簡單。接下來是重點。

Part B: Expected Value期望值

💡 The one idea you must understand first:

Expected value = a long-term average. It is NOT any single outcome.

期望值 = 長期平均，不是某一次的結果。

Let the dice convince you讓骰子說服你

Throw a fair dice once. What number do you get?
擲一粒公平骰子一次，你會得到什麼數字？

1
2
3
4
5
6

You could get any one of these — each with probability 1/6.
你可能得到任何一個 —— 每個概率都是 1/6。

Now throw the dice 600 times. What would the average of all 600 results be?
現在擲 600 次。所有結果的平均值會是多少？

1's:
≈ 100 times 約100次
2's:
≈ 100 times 約100次
3's:
≈ 100 times 約100次
4's:
≈ 100 times 約100次
5's:
≈ 100 times 約100次
6's:
≈ 100 times 約100次

Total sum ≈ 1×100 + 2×100 + 3×100 + 4×100 + 5×100 + 6×100 = 2100
Average = 2100 ÷ 600 = 3.5

總和 ≈ 2100，平均值 = 3.5

3.5 is the average — it is not a number you can ever roll!

You will never throw a dice and see "3.5" come up. But if you throw it thousands of times,
the average of all your results will be very close to 3.5.

3.5 是長期平均值 —— 你永遠不會擲出 "3.5"，但擲很多次後，所有結果的平均會非常接近 3.5。

The formula — and why it works公式及其原理

Expected Value = x₁p₁ + x₂p₂ + … + xₙpₙ

期望值 = 每個可能結果的值 × 其概率，全部加總

What this formula actually does: it weights each outcome by how likely it is.
這個公式其實在做的事：用概率加權每個結果。

A fair dice is thrown once. Find the expected value of the number obtained.
擲一粒公平骰子一次，求所得數字的期望值。

Number obtained

Part A vs Part B — don't mix them up兩個公式，不要搞混

Part A
預期次數

n × p

"How many times will this happen?"
「這件事會發生幾次？」
One event × many trials
一個事件 × 多次試驗

Part B
期望值

Σ xᵢpᵢ

"What's the average value I can expect?"
「長期平均下來，我可以期望多少？」
Each outcome × its probability, summed
每個結果 × 其概率，全部相加

Decision Clue: Is the game favourable?判斷遊戲是否有利

Example 3 (from textbook)教材例題 3

A bag contains 1 red ball, 3 yellow balls and 8 orange balls. A player draws one ball at random. 一個袋裡有 1 個紅球、3 個黃球、8 個橙球。玩家隨機抽一個球。

Step-by-step: How to build the table yourself如何自己建立表格

1
Count total possible outcomes. How many balls in total?
先數總共有多少個可能結果。
1 (red) + 3 (yellow) + 8 (orange) = 12 balls total
紅 + 黃 + 橙 = 12 個球
2
For each outcome, write its value and calculate its probability.
對每一個結果，寫下它的值，並計算概率。
Outcome結果Value ($)值Count數量Probability = Count ÷ Total概率
Red紅球3011/12
Yellow黃球2033/12 = 1/4
Orange橙球688/12 = 2/3
3
Apply the formula: each value × its probability, then add them all.
套公式：每個結果的值 × 其概率，全部相加。
EV = 30×(1/12) + 20×(1/4) + 6×(2/3) = $11.5

⚠️ Common mistake: forgetting to divide the count by the total. 1/12 not just 1. 3/12 not just 3. Always: probability = count ÷ total.
常見錯誤：忘記把數量除以總數。概率永遠是「數量 ÷ 總數」，不是只用數量。

期望獎金 = 每個獎金 × 對應概率，全部相加 = $11.5

(b) If the cost of playing the game once is $10, is the game favourable?
如果玩一次的成本是 $10，這個遊戲對玩家有利嗎？

The Decision Clue判斷法則

Cost < EV
成本 < 期望值
👍 Favourable
有利
In the long run, you gain.
Cost = EV

🤝 Fair

Cost > EV

For Example 3: Cost = $10, EV = $11.5

$10 < $11.5 → Cost < EV → The game is favourable to the player. 🎉

成本 $10 < 期望值 $11.5 → 有利於玩家

This means: if you play this game many times, on average you will earn $1.5 per game.
意思是：如果你玩很多次，平均每次你會賺 $1.5。

Try it yourself自己試試

① In a game, a player tosses a fair coin once. If a head is obtained, the player gets $50. Otherwise, the player gets $10. Find the expected value. 擲一次公平硬幣：正面得 $50，反面得 $10。求期望值。
P(Head) = 1/2, P(Tail) = 1/2
EV = $50 × (1/2) + $10 × (1/2) = $25 + $5 = $30 期望值 = 50×(1/2) + 10×(1/2) = $30
② There are 9 red balls, 7 yellow balls and 4 blue balls in a bag. Jeff randomly draws a ball. 10 tokens for red, 50 for yellow, 100 for blue. (a) Find the expected number of tokens. (b) If the cost is 40 tokens, is the game favourable? 袋中有 9 紅、7 黃、4 藍球。紅=10代幣、黃=50、藍=100。求期望值，若成本=40，是否有利？
Total balls = 9+7+4 = 20
P(Red) = 9/20, P(Yellow) = 7/20, P(Blue) = 4/20

(a) EV = 10×(9/20) + 50×(7/20) + 100×(4/20)
= 4.5 + 17.5 + 20 = 42 tokens

(b) Cost = 40, EV = 42 → Cost < EV → Favourable ✅ 期望值 = 42，成本 40 < 42 → 有利
③ Why is the expected value of a dice throw 3.5 even though you can never roll a 3.5? 為什麼擲骰子的期望值是 3.5，但你永遠不會擲出 3.5？
Expected value is a long-term average, not a single outcome.
If you roll a dice 6000 times, the average of all results will be approximately 3.5 — but no single roll can ever be 3.5. It's like saying "the average family has 2.4 children" — no family literally has 2.4 children, but the average across all families is 2.4. 期望值是長期平均，不是某一次的結果。就像「平均每個家庭有 2.4 個孩子」——沒有家庭真的有 2.4 個孩子，但所有家庭的平均確實是 2.4。
